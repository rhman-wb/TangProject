## 1. 项目背景

《全唐诗》共收诗词四万九千余首，其中将近一半在长安写成或与长安有关；诗词作者二千二百余人，一半以上是长安诗人或曾在长安居住、工作过的诗人。诗人咏长安，为我们展开了一幅完整的历史画卷。

本项目旨在通过**交互式地图**的形式，将唐代诗人在长安（今西安）创作的诗歌、诗句所描绘的地点进行可视化标注，图文并茂地绘制一幅"唐诗地图"，让用户在地图上直观地探索诗词背后的故事，让千年诗词在数字时代继续焕发活力。

项目参考开源项目 worldhistoryviz 的设计思路与技术框架：以 Leaflet.js 为地图引擎、以结构化 JSON 为数据载体、以纯前端静态站点的形态构建，可零成本部署与传播。

## 2. 项目目标

### 2.1 产品目标

1. 在西安（长安）地图上准确标注唐诗的**创作地点**或**诗句描绘的地点**，每个地点关联诗名、诗人、名句、背景故事与配图。
2. 覆盖需求指定的 25 位唐代诗人（初唐 6 人、盛唐 4 人、中唐 8 人、晚唐 7 人），首期每位诗人至少收录 2 首与长安相关的代表诗作，总量不少于 60 首。
3. 提供按**时期**（初唐/盛唐/中唐/晚唐）、**诗人**、**地点类型**的筛选与检索能力，以及可拖动的**时间轴**浏览体验。
4. 界面美观、具有唐风美学特征，桌面端与移动端均可流畅使用。

### 2.2 成功衡量指标

| 指标 | 目标值 |
| --- | --- |
| 收录诗词数量 | ≥ 60 首（首期） |
| 标注地点数量 | ≥ 30 处 |
| 首屏加载时间 | ≤ 3 秒（普通 4G 网络） |
| 移动端适配 | 完整支持 375px 及以上宽度 |
| 数据准确性 | 诗文、作者、地点考据准确，重要信息附出处 |

## 3. 目标用户

| 用户画像 | 场景与诉求 |
| --- | --- |
| 中小学生 / 暑期研学学生 | 结合"探寻身边的长安诗词"研学任务，在地图上寻访诗歌地点，了解诗词背后的故事 |

## 4. 收录范围

### 4.1 诗人名录（25 人）

| 时期 | 诗人 |
| --- | --- |
| 初唐（618–712） | 王勃、杨炯、卢照邻、骆宾王、杜审言、陈子昂 |
| 盛唐（712–762） | 王之涣、王维、李白、杜甫 |
| 中唐（762–835） | 白居易、元稹、韩愈、柳宗元、刘禹锡、李贺、韦应物、卢纶 |
| 晚唐（835–907） | 李商隐、杜牧、温庭筠、韦庄、皮日休、陆龟蒙、罗隐 |

> 说明：经评审（见 ADR-0001），时期改用初/盛/中/晚**四段**（边界 618 / 712 / 762 / 835 / 907），将原"中唐（含盛唐）"中的盛唐四位诗人（王之涣、王维、李白、杜甫）单列，以符合文学史、避免"李白·中唐"之类的误标。数据层统一为单一 `period` 字段（early/high/middle/late），不再保留冗余的"文学史时期"字段。

### 4.2 地点范围

地图范围分**核心**与**扩展**两层（见 ADR-0006），由地点数据的显式 `scope` 字段（`core` / `extended`）人工策展判定，不依赖坐标自动判断：

- **核心范围（core）**：唐长安城内坊市宫苑及京畿近郊，默认显示。包括但不限于：
  - **城内坊市宫苑**：大明宫、兴庆宫、太极宫、朱雀大街、平康坊、曲江池、乐游原、大雁塔（大慈恩寺）、小雁塔（荐福寺）、青龙寺等；
  - **近郊胜迹**：灞桥（灞柳送别）、渭城（咸阳）、终南山、华清宫（骊山）、香积寺、辋川（蓝田）、樊川（杜曲）等；
- **扩展范围（extended）**：长安以外但与长安主题强相关的远地（如送别诗目的地阳关/安西、马嵬、巴蜀等），**默认隐藏**，由顶部「扩展范围」开关显示。

> 注：`scope`（地点的地理归属）与 `placeRelation`（诗↔地语义：创作/描绘/关联）是**正交**的两个维度。例如马嵬 `scope=extended`，但在《长恨歌》中 `placeRelation=described`。
>
> 另：不可定位的文学/虚构地点（如《长恨歌》仙界、《梦游天姥吟留别》梦境）v1 不收录为地点、不建标记，仅在背景故事中说明。

### 4.3 单条诗词数据要求

每首入选诗词须包含：诗名、作者、时期（初/盛/中/晚）、全文、**名句**（可多句）、创作/描绘地点（含坐标与核心/扩展归属）、地点古今对照说明、背景故事、配图、资料出处。各字段按 v1 数据分级分为"强校验必填"与"最小可用"两档（见 §6.3、ADR-0007）；配图首期用占位符、后续以 AI 生图批量替换（见 ADR-0008）。

### 5.2 功能详述

#### F1 交互式地图（P0）

- 以西安市为中心初始化地图，默认中心点为唐长安城核心区（约 108.94°E, 34.27°N），默认缩放级别覆盖长安城及近郊。
- 支持鼠标/触摸的缩放、平移；提供"回到长安"复位按钮。
- 底图采用可免费商用的瓦片服务，中文标注优先。
- 限制最大浏览范围为关中地区，防止用户迷失；最大范围（maxBounds）默认限制在核心关中区域，「扩展范围」开关打开时放宽到含扩展地点的大范围、关闭时收回。

#### F2 诗词地点标记（P0）

- 每个地点在地图上显示自定义样式标记（唐风设计，如印章/毛笔点样式）。
- 标记颜色按时期区分：初唐、盛唐、中唐、晚唐各一色（色板见设计文档）。
- 「扩展范围」关闭时仅显示 `scope=core` 的标记；打开后追加 `scope=extended` 的标记（淡入过渡）。
- 同一地点关联多首诗词时，标记显示数量角标，点击后在详情面板中列出该地点的全部诗词。
- 地图缩小时对邻近标记进行**聚合**（cluster），避免重叠。
- 鼠标悬停（桌面端）显示快捷气泡：诗名 + 诗人 + 一句名句。
- 点击标记：地图平滑移动至该点，打开详情面板。

#### F3 诗词详情面板（P0）

- 桌面端为右侧滑出面板，移动端为底部上滑抽屉。
- 内容结构：
  1. 地点名称（古称 + 今称，如"曲江池 · 今曲江池遗址公园"）；
  2. 地点配图（支持多图轮播）；
  3. 诗词卡片：诗名、作者（可点击跳转诗人页）、时期徽章、全文（竖排或横排可选）、名句高亮；
  4. 背景故事：创作背景、诗句与地点的关联考据；
  5. 资料出处。
- 全文展示需支持繁简通用的中文排版，长诗（如《长恨歌》）支持折叠/展开。

#### F4 时期/诗人筛选（P0）

- 顶部或侧栏提供"初唐 / 盛唐 / 中唐 / 晚唐 / 全部"分段筛选器。
- 诗人筛选：按时期分组的诗人清单（复选），选中后地图仅显示对应诗人的标记。
- 筛选状态与地图标记、列表侧栏实时联动。

#### F5 诗词列表侧栏（P0）

- 左侧（桌面）/抽屉（移动）展示当前筛选条件下的诗词列表，每项显示：诗名、诗人、地点、名句摘录。
- 点击列表项：地图飞行定位到对应标记并打开详情。一首诗关联多个地点时，列表仍为一项，飞行目标为其首个核心范围内（`scope=core`）的地点（无则取首个），该诗其余地点的标记做高亮提示。
- 列表支持按时期、诗人、地点分组排序。

#### F6 搜索（P1）

- 单一搜索框，支持对诗名、诗人名、诗句正文、地点名的模糊匹配。
- 输入时实时下拉提示（防抖 300ms），结果分类显示（诗词/诗人/地点）。

#### F7 时间轴（P1）

- 页面底部横向时间轴，范围 618–907 年，标注四个时期分界（618 / 712 / 762 / 835 / 907）。
- 拖动游标或点击"播放"按钮，按创作年代逐步显示/隐藏标记；创作年不可考者取诗人主要活动年代中值，但 UI 须诚实标注（如"约 X 年 / 活动年代"），避免让用户误以为是确切创作年（v1 沿用单 `year` 字段方案）。
- 时间轴与筛选器逻辑取交集。

#### F8 诗人介绍页（P1）

- 每位诗人一个介绍视图：头像/画像、生卒年、字号、时期、小传（200 字内）、与长安的渊源、收录作品列表。
- 从详情面板中的作者名、诗人筛选列表均可进入。

#### F9 古今地图对照（P1）

- 提供"唐长安城复原图"图层开关：将长安城坊、宫城、皇城轮廓（GeoJSON 多边形）半透明叠加在现代地图之上。
- 坊名可在高缩放级别下显示注记。

#### F10–F12（P2）

- 主题切换：浅色/暗色两套配色，默认跟随系统 `prefers-color-scheme`。
- 分享：URL 携带 `?poem=<id>` / `?place=<id>` 参数，打开时自动定位并展开详情。
- 引导页：首次访问展示项目简介与三步操作引导，可跳过，localStorage 记忆。

## 6. 数据需求

### 6.1 数据实体

| 实体 | 说明 | 预估规模（首期） |
| --- | --- | --- |
| 诗人 poets | 25 位诗人的基本信息 | 25 条 |
| 地点 places | 长安及周边诗词地点（含坐标、古今名、核心/扩展 `scope`） | 30–50 条 |
| 诗词 poems | 诗词全文、名句（可多句）、关联诗人与地点、背景故事 | 60–100 条 |
| 长安城复原图层 | 城坊/宫城轮廓 GeoJSON | 1 套 |

### 6.2 数据生产流程

1. **资料调研**：以《全唐诗》、权威赏析资料及地方志为依据，人工/AI 辅助梳理每位诗人与长安相关的代表作。
2. **地点考据**：确定创作地或描绘地的今日坐标（精确到具体遗址/公园/街区），不可精确考据者标注"约略位置"并在详情中说明；并标注该地点为核心（core）还是扩展（extended）范围。
3. **结构化录入**：按设计文档定义的 JSON Schema 录入，字段校验通过后合入数据目录。
4. **审校**：诗文以权威版本校对；坐标在地图上人工核验。

> 生产方式（见 ADR-0007）：AI 辅助起草（诗文/作者以权威版核对，故事与坐标 AI 起草）+ 人工校验；按字段分级，强校验必填项不达标不予合入。配图首期统一占位符（附生图 prompt），后续以 Grok 生图模型按统一唐风美术方向批量替换（见 ADR-0008）。

### 6.3 数据质量要求

- 诗词全文与作者归属须准确（强校验），存疑作品不收录或明确标注争议；
- **v1 数据分级**：诗名 / 作者 / 时期 / 全文 / 名句 / 坐标 / 出处为**强校验必填**（完整率 100%）；背景故事与古今对照说明为**最小可用**（v1 可简、后续迭代补全）；
- 配图首期使用占位符，后续以 Grok 生图模型按统一唐风美术方向批量生成（见 ADR-0008），从根本上规避版权风险。

## 9. 风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| 地点考据困难，部分诗词创作地不可考 | 数据准确性 | 采用"描绘地点"替代"创作地点"；标注"约略位置"；详情中说明考据依据 |
| 免费底图服务的可用性/配额限制 | 地图无法加载 | 选择多家瓦片源并支持配置切换（详见设计文档） |
| 配图版权 | 法律风险 | 首期占位符上线，后续以 Grok 生图模型批量生成（统一唐风方向），规避版权风险（ADR-0008） |
| 数据量增长导致首屏变慢 | 性能 | JSON 按实体分文件、按需加载；标记聚合渲染 |
| 唐长安城复原图数据获取 | F9 可能延期 | 该功能为 P1，可基于学术复原图自行绘制简化 GeoJSON，或降级为静态示意图 |

## 10. 验收标准

1. 地图正常加载并可缩放平移，所有 P0 功能可用且联动正确；「扩展范围」开关可正常显隐远地标记；
2. 收录诗词 ≥ 60 首（按数据分级，强校验字段 100% 完整）、地点 ≥ 30 处；时期为初/盛/中/晚四段分组；抽检 20% 数据无诗文/作者/坐标错误；
3. 桌面端与移动端（375px 宽）核心流程（浏览 → 筛选 → 点击标记 → 阅读详情）均可顺畅完成；
4. Lighthouse Performance ≥ 80、Accessibility ≥ 90；
5. 成功部署至公网可访问的静态站点地址。

## 附录 A：术语

| 术语 | 释义 |
| --- | --- |
| 标记（Marker） | 地图上代表一个诗词地点的可点击图标 |

| 坊 | 唐长安城的基本居住区划单位，全城约 108 坊 |
