# 一诗多地用 places 关联数组

一首诗关联多个地点时，`poem` 以 `places: [{ placeId, relation }]` 关联数组承载——每项既指明地点、又指明该地与诗的关系（`composed` / `described` / `related`），渲染层为每个地点各建一枚标记，而不是把同一首诗拆成多条记录。这样诗作的全文与元数据保持单一权威来源，避免重复录入与版本不一致；多地点的地图呈现完全交给标记层处理，且一诗多地的不同关系也能各自表达（如《送元二使安西》：渭城 described、阳关 related）。

> 此结构取代早期的 `placeId` 数组 + 标量 `placeRelation` 方案——标量无法表达一诗多地的不同关系。
